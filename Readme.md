# mServer — Python-сервер для моделирования РЛС

`mServer` — UDP-сервер на Python, реализующий вычислительное ядро для моделирования сигналов радиолокационной станции (РЛС). Работает в связке с клиентом `SamRLSim` (MATLAB/Simulink): клиент отправляет команды и параметры, сервер выполняет физически корректные расчёты и возвращает результат.

---

## Архитектура

```
SamRLSim (MATLAB/Simulink)
        │  UDP :9092 ←─── ответы
        │  UDP :9098 ───→ команды
        ▼
   mServer (Python)
   ├── main.py          — точка входа, цикл приёма команд
   ├── state.py         — глобальный AppState (синглтон)
   ├── settings/
   │   ├── server.py    — конфигурация сокетов
   │   └── models.py    — датаклассы: Rs, Mi, Tr, St, Sf, Sea, consts
   ├── helpers/
   │   ├── parse_msg.py — парсинг входящих сообщений
   │   ├── params.py    — применение параметров к AppState
   │   └── format_error.py
   └── scripts/
       ├── get_surface/ — построение поверхности (рельеф, море, лес)
       ├── get_traekt/  — расчёт траектории цели
       ├── get_mixyz/   — геометрия многоэлементного излучателя (МИ)
       ├── do_sign_mod/ — предрасчёт сигнала (ФМ, FM-модуляция)
       └── do_step/     — расчёт относительных мощностей излучателей
```

### Протокол обмена

Каждое сообщение от клиента — строка с набором переменных и команд:

```
VarName1=value1; VarName2=value2; ... CommandName1; CommandName2
```

Сервер применяет переменные к `AppState`, затем выполняет перечисленные команды последовательно и отправляет ответ на каждую.

| Команда        | Описание                                   |
|----------------|--------------------------------------------|
| `Get_Surface`  | Построение поверхности фона                |
| `Get_Traekt`   | Расчёт траектории цели                     |
| `Get_MiXyZ`    | Координаты и геометрия антенн МИ           |
| `Do_SignMod`   | Предрасчёт сигнала (модуляция)             |
| `Do_Step`      | Расчёт мощностей излучателей               |
| `ans`          | Версия сервера                             |
| `server time`  | Время работы сервера (uptime)              |
| `exit`         | Остановка сервера                          |

## Модули расчёта

### `Get_Surface` — фоновая поверхность

Формирует матрицу `cMass` — координаты фасетов поверхности. Поддерживаемые типы:

| Код | Тип      | Описание                     |
|-----|----------|------------------------------|
| 10  | ground   | Грунт                        |
| 20  | water    | Вода (морская поверхность)   |
| 30  | meadow   | Луг / трава                  |
| 40  | forest   | Лес                          |
| 50  | snow     | Снег / скалы                 |
| 60  | sand     | Песок / солончак             |
| 70  | road     | Дорога / город               |
| 80  | bushes   | Кустарник / прочее           |

**Морская поверхность** (`get_sea.py`) моделируется через TMA-спектр (при скорости ветра ≥ 4 м/с) или спектр капиллярных волн (< 4 м/с). Параметры волнения определяются по шкале Бофора (1B–9B) в зависимости от скорости ветра от 0 до ≥ 20 м/с. Рассчитываются: длины волн `L`, волновые числа `K`, угловые частоты `W`, случайные начальные фазы `Fi0`.

**Лесная поверхность** (`get_forest.py`) параметризуется типом леса (хвойный, лиственный, без листвы, смешанный, многоярусный) и плотностью (%).

### `Get_Traekt` — траектория цели

Рассчитывает временну́ю последовательность координат и углов цели `Tr` (объект `Tr`) на основе начального положения `(Xa, Ya, Za)`, скоростей `(Vx, Vy, Vz)` и ориентации (крен, тангаж, рыскание).

### `Get_MiXyZ` — геометрия МИ

Вычисляет координаты 4 излучателей многоэлементного излучателя в пространстве (`Mx`, `My`, `Mz`), угловые координаты `(Ax, Az)` и расстояния до центра АС РЛС. Используется для расчёта выравнивающих кабелей (`Diffs`, `dDist`).

### `Do_SignMod` — модуляция сигнала

Поддерживаемые виды сигналов:
- **Простой импульс** (`do_sign_imp.py`)
- **ФМ (фазовая манипуляция)** (`do_sign_fm.py`) — последовательности SC1, SC5 и др.
- **FM (линейная частотная модуляция)** (`do_sign_mod.py`)

### `Do_Step` — шаг моделирования

Рассчитывает относительные мощности каждого из 4 излучателей МИ (`Mi.Pi`) на текущем шаге по времени `Step`. Обновляет `state.t = Step * 0.001`.

## Структура состояния (AppState)

Глобальный объект `state` (`state.py`) хранит всё состояние симуляции:

| Поле     | Тип        | Описание                                    |
|----------|------------|---------------------------------------------|
| `Rs`     | `Rs`       | Параметры РЛС (импульсы, поляризация, АРУ)  |
| `Mi`     | `Mi`       | Геометрия и мощности МИ                     |
| `Tr`     | `Tr`       | Траектория цели                             |
| `St`     | `St`       | Параметры сцены / статических объектов      |
| `Sf`     | `Sf`       | Параметры поверхности и рельефа             |
| `Sea`    | `Sea`      | Параметры морского волнения                 |
| `consts` | `consts`   | Физические константы (g, π, δ)              |
| `t`      | `int`      | Текущее время симуляции                     |
| `Step`   | `int`      | Текущий шаг                                 |
| `cMass`  | `ndarray`  | Матрица координат фасетов поверхности       |
| `c`      | `int`      | Скорость света, 299 792 458 м/с             |

Вспомогательная функция `evs(expr)` вычисляет строковые выражения в контексте `state` с поддержкой MATLAB-синтаксиса (`.*`, `./`, `.^`, `^`).

## Настройка проекта

### Требования

- Python 3.10+
- Клиент `SamRLSim` расположен в той же родительской директории, что и `mServer`

### Установка

```bash
# Переместить директорию SamRLSim и mServer в одну папку,
# затем в корне mServer:

python -m venv .venv
.venv\Scripts\activate       # Windows
# source .venv/bin/activate  # Linux/macOS

pip install -r requirements.txt
```

### Зависимости

| Пакет        | Назначение                                |
|--------------|-------------------------------------------|
| `numpy`      | Векторные и матричные вычисления          |
| `scipy`      | Преобразования поворота (Rotation)        |
| `matplotlib` | Визуализация поверхности и рельефа        |
| `pillow`     | Сохранение результатов в BMP/PNG          |
| `pywin32`    | Работа с буфером обмена Windows           |
| `ruff`       | Линтер / форматтер                        |

## Запуск

```bash
python main.py
```

Сервер запускается на `localhost:9098` (приём) и отправляет ответы на `localhost:9092`.  
При успешном старте выводится:

```
UDP server running with port 9098
```

Остановка — командой `exit` от клиента или `Ctrl+C`.
